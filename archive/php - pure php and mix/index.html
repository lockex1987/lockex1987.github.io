Sử dụng PHP thuần với Laravel Mix

Tôi có một dự án, trong đó chia thành hai phần frontend và backend riêng biệt. Frontend sử dụng PHP Laravel, dạng SPA với Vue. Backend do đội khác phát triển bằng Java Spring. Frontend không có xử lý logic gì bằng PHP, do đó nếu dùng cả framework Laravel thì có vẻ thừa nhiều quá.

Tuy nhiên, tôi vẫn muốn một số tính năng mà Laravel cung cấp là:

  • Lấy cấu hình từ file .env
  • Đường dẫn đến các tài nguyên CSS, JS được thêm các hash để đảm bảo lấy file mới nhất (bằng cách sử dụng hàm mix() của Laravel)

Tôi quyết định viết lại bằng PHP thuần mà vẫn có hai tính năng trên.

File mẫu:

platform_index.php

Cấu hình route ở nginx cho Laravel đã đáp ứng luôn rồi.

Chú ý, ở file webpack.mix.js, bạn phải có cấu hình:

mix.setPublicPath('public');

Mix có đoạn kiểm tra, nếu là project Laravel (có file artisan) thì sẽ tự động cấu hình đường dẫn publicPath là public.

Bạn có thể sinh ra file HTML từ file PHP trên bằng lệnh:

php index.php > index.html
